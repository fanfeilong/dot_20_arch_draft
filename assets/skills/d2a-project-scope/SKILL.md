# d2a-project-scope

## Goal

Clarify the minimum project boundary before deeper architecture analysis.

## Instructions

1. Identify the project's primary runtime or execution environment.
2. Name the one to three highest-value entry points.
3. State what should be treated as in-scope and out-of-scope.
4. Keep the output short and structural, not historical.

## Output

- Runtime
- Entry points
- In scope
- Out of scope
