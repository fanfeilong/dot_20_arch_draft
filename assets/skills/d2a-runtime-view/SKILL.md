# d2a-runtime-view

## Goal

Describe the runtime shape of the target project with minimum words.

## Instructions

1. Identify the main process or service boundaries.
2. List the critical runtime components.
3. Describe how requests, events, or jobs move across them.
4. Prefer a compact structural view over explanation.

## Output

- Processes
- Core components
- Main interactions
- One dominant runtime path
