# d2a-module-view

## Goal

Split the codebase into a small number of architecture-relevant modules.

## Instructions

1. Group files or packages by architectural responsibility.
2. Avoid fine-grained file-by-file narration.
3. Surface only the modules that matter to understanding the core.
4. Keep each module description to one short statement.

## Output

- Module name
- Responsibility
- Main dependencies
- Why it matters
